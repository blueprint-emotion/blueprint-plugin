# Wireframe Design Guide

와이어프레임은 "예쁜 화면"이 아니라 구조와 위계를 빠르게 검토하는 산출물이다.
FlowFrame용 와이어프레임은 장식보다 정보 구조, 관계, 작업 흐름이 먼저 보여야 한다.

## 핵심 원칙

1. 콘텐츠를 주인공으로 둔다. 헤더, 필터, 보조 패널은 메인 콘텐츠보다 시각적으로 약해야 한다.
2. 색보다 타이포와 간격으로 위계를 만든다. 유채색은 쓰지 않거나 최소화한다.
3. 정보 밀도를 유지하되 숨 막히지 않게 만든다. 요소 간 여백은 충분히 주되, 큰 빈 공간은 남기지 않는다.
4. 장식을 배제한다. 강한 그림자, 그라데이션, 과한 라운드, 화려한 배경은 쓰지 않는다.

## 시각 규칙

- 기본 텍스트는 `text-sm`를 사용한다.
- 제목은 `text-lg` 또는 `text-xl`까지만 사용한다. `font-bold`는 쓰지 말고 `font-semibold`까지만 사용한다.
- 레이아웃 위계는 `background`/`card` 수준의 무채색 대비와 간격으로만 만든다.
- 카드, 패널, 입력 필드, 버튼은 단순한 박스 형태로 유지한다.
- 모든 색상 관련 클래스에는 `dark:` 변형을 포함한다.

## 간격 규칙

- 좁은 그룹 내부는 `gap-2` 또는 `gap-3`
- 카드 내부 패딩은 `p-4`
- 패널 내부 패딩은 `p-4` 또는 `p-6`
- 섹션 간 간격은 `gap-6` 이상
- 레이아웃이 느슨해 보이면 먼저 높이를 줄이고, 여백은 유지하되 큰 공백을 제거한다

## 컨텐츠 폭 기준

- PC 와이어프레임은 메인 콘텐츠 최대 너비를 `1200px` 기준으로 잡는다.
- 모바일 와이어프레임은 메인 콘텐츠 너비를 `370px` 기준으로 잡는다.
- PC에서는 `mx-auto`로 가운데 정렬하고, 좌우 패딩을 주더라도 실제 콘텐츠 폭은 `1200px`을 크게 넘기지 않는다.
- 모바일에서는 화면 전체를 다 쓰기보다 `370px` 안에서 카드, 입력 필드, 버튼이 안정적으로 보이게 구성한다.
- 멀티 뷰포트 화면은 각 뷰포트별 HTML에서 이 기준 폭을 따로 적용한다.

## 레이아웃 규칙

### 검색 + 필터 + 리스트/그리드 화면

- 검색 영역은 한 줄에서 빠르게 읽혀야 한다.
- 검색 버튼은 너무 좁아지지 않게 고정 폭 또는 충분한 패딩을 준다.
- 필터 패널은 메인 콘텐츠보다 좁아야 한다. 보통 `w-64`~`w-72` 수준으로 제한한다.
- 필터 패널 높이를 억지로 늘리지 않는다. 내용만큼만 차지하게 둔다.
- 메인 리스트/그리드는 화면의 시각적 중심이 되어야 한다.

### 카드형 목록 화면

- 카드 이미지는 카드 높이의 절반을 넘기지 않는다.
- 카드 안의 텍스트 정보는 3~4줄 정도로 유지한다.
- CTA 버튼은 카드 하단에서 충분한 너비와 높이(`h-10` 이상)를 가져야 한다.
- 반복 카드가 너무 헐거워 보이면 이미지 높이보다 정보 밀도와 카드 간격부터 조정한다.

### 폼 화면

- 입력 필드와 버튼은 같은 리듬으로 정렬한다.
- 보조 링크는 주 CTA보다 약하게 보여야 한다.
- 한 화면에 CTA는 하나만 강조한다.

## 안티 패턴

- 검색 버튼이 입력창 옆에서 지나치게 작거나 눌려 보이는 구성
- 사이드바/필터 패널이 세로로 과도하게 길어져 빈 공간이 크게 남는 구성
- 플레이스홀더 박스가 실제 정보보다 더 눈에 띄는 구성
- 카드 이미지가 너무 커서 텍스트와 액션이 밀리는 구성
- 의미 없는 그림자, 배경 장식, 다중 포인트 컬러 사용
- 요소 간 간격이 제각각이라 화면 리듬이 깨지는 구성

## 품질 체크

출력 전에 아래를 확인한다.

- 메인 콘텐츠가 첫눈에 중심으로 보이는가
- 검색, 필터, 카드, CTA의 비율이 자연스러운가
- 긴 빈 공간이 남지 않았는가
- 반복 카드의 밀도와 정렬이 안정적인가
- 와이어프레임답게 단순하지만, 정보 구조는 분명한가
