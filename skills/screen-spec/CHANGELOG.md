# screen-spec — Changelog

이 스킬의 변경 이력. SemVer(MAJOR.MINOR.PATCH)를 따른다.

- **MAJOR**: 파일명 패턴·frontmatter 필드·본문 구조의 호환성 깨는 변경
- **MINOR**: 새 type·새 선택 필드·새 패턴 추가 (호환성 유지)
- **PATCH**: 문서 명료화·예제 보강·오타 수정

## [1.6.1] - 2026-04-20

### Added

- `user-invocable: false` frontmatter — 슬래시 명령(`/bp:screen-spec`) 노출 방지. planner agent에서만 자동 트리거

## [1.6.0] - 2026-04-20

### Removed

- `## 접근성` 섹션 (루트 화면명세·영역 화면명세 모두) — 접근성은 개발 표준이지 화면별 기획 결정이 아님. 화면별 특수 사항(자동 포커스 타겟, aria 카피, 단축키)은 `## 인터랙션`이나 elements에 흡수. 기본 KWCAG 준수는 사내 코딩 컨벤션으로 분리
- "### 접근성 섹션 작성법" 가이드 섹션 제거
- "자주 하는 실수" 표에서 "접근성 섹션 생략" 행 제거
- description 본문·키워드에서 "접근성" 제거
- references/area_image.md, references/area_option.md 예제에서 `## 접근성` 섹션 제거

### Background

이전에는 "기획 단계에서 접근성을 명시해야 개발 종료 직전 재작업을 막을 수 있다"는 이유로 `## 접근성`을 옵셔널 섹션으로 두었으나, 실제 작성 시 화면별로 결정할 게 없어 계속 일반론(KWCAG 체크리스트)만 복붙되는 패턴이 발견됨. 화면 고유 사항(자동 포커스, aria 카피)은 인터랙션/elements와 책임이 겹쳐 어느 쪽에 적을지 모호. 결과적으로 SSOT 위반 + 빈 섹션이 되는 경우가 많아 제거 결정.

### Migration

기존 화면명세에 `## 접근성` 섹션이 있어도 호환성에 문제 없음 (단지 새로 작성 시 만들지 않을 뿐). 화면별 자동 포커스·키보드 단축키·aria 카피 결정 사항은 `## 인터랙션`이나 elements 본문으로 옮길 것을 권장.

## [1.5.2] - 2026-04-15

### Added
- "elements 작성법" 시작에 **형식 표** 추가 — `**elements**`는 `##` 헤딩이 아니라 bold 인라인 라벨임을 명시. 영역 파일·단순 화면 인라인 두 위치별 배치 규칙 표로 정리
- 영역 본문 섹션 표의 `**elements**` 행에 "(bold 라벨, `##` 아님)" 부기 + 위치 설명 보강

### Background
- 검증 에이전트가 SKILL의 elements 형식 표기와 references의 실제 사용(`**elements**` 라벨 + 블릿) 사이에 미세한 모호함을 발견. 모순은 아니지만 처음 읽는 사람이 헤딩으로 오해할 수 있어 명문화

## [1.5.1] - 2026-04-15

### Fixed
- 우선순위 라벨 위치 — "유저스토리 뒤에" → "유저스토리 **앞에**"로 정정 (예시는 처음부터 앞에 표기되어 있어 본문과 모순이었음). grep·정렬 용이성 측면에서도 앞이 맞음
- "화면 구성 작성법" 서브섹션 위치 이동 — 본문 표 직후로 (인터랙션·예외·접근성 작성법보다 앞). 본문 표에서 두 번째 필수 섹션이므로 가까이 두는 게 자연스러움

### Added
- 본문 표 아래에 **"데이터 갱신 주기는 feature-spec에 둔다"** 가이드 한 줄 추가 — `## 예외 케이스`(외부 시스템 실패)와 데이터 생명주기(도메인 갱신 정책)의 역할 구분 명시
- 단일 영역 화면 인라인 작성 항목에 "인터랙션·예외 케이스·접근성도 같은 파일에 인라인" 보강
- description에 새 섹션 키워드 (`인터랙션`, `예외 케이스`, `접근성`, `우선순위`) 추가 — 자동 트리거 정확도 향상

## [1.5.0] - 2026-04-15

옵셔널 섹션 3개 + 유저스토리 우선순위 도입.

### Added
- `## 인터랙션` 섹션 (루트·영역 모두 옵셔널) — 모바일 제스처·키보드·자동 포커스 등 Web Component 외형에 드러나지 않는 행동 규칙. `구성요소 / 인터랙션 / 상세` 표 형식
- `## 예외 케이스` 섹션 (루트·영역 모두 옵셔널) — 외부 시스템·환경 실패(API 오류·네트워크 오프라인·비로그인·권한 없음·영업시간 외). `상황 / 처리 방식` 표. `## 상태`(UI 뷰)와 역할 구분 명시
- `## 접근성` 섹션 (루트·영역 모두 옵셔널) — 스크린리더·키보드·포커스·색대비·이미지 alt 체크리스트. 한국 웹접근성 지침(KWCAG) 기준. PC/MO 차이 있으면 서브헤딩으로 분리
- 유저스토리 **우선순위 라벨** (`[Must]`/`[Should]`/`[Could]`) — 각 유저스토리 항목에 표기. 릴리스 범위 협상·백로그 정렬 근거
- 자주 하는 실수 4항목:
  - 유저스토리에 우선순위 누락
  - `## 상태`에 예외 케이스 섞어 쓰기
  - 인터랙션을 와이어프레임만 보고 유추하라고 생략
  - 접근성 섹션 생략

### 참고
- 데이터 갱신 주기는 화면마다 중복 서술하지 않는다. **feature-spec v1.4.0의 `## 데이터 생명주기` 섹션**에 도메인 단위로 두고, screen-spec은 features[] 링크로 자동 연결

## [1.4.0] - 2026-04-15

### Added
- 루트 `screen.md` 본문 섹션 표에 `## 유저스토리` 추가 — 다중 영역 화면은 선택(화면 전체의 큰 목표 1~2개만), 단일 영역 화면은 필수(영역 파일이 없어 인라인으로 써야 함)
- "유저스토리 레벨 분담" 서브섹션 신설 — 루트는 *"이 화면을 통해 ~하고 싶다"*, 영역은 *"이 영역에서 ~하고 싶다"* 템플릿으로 스케일 구분
- 자주 하는 실수 2항목 추가:
  - 루트와 영역에 같은 유저스토리 중복 작성
  - 단일 영역 screen.md에 유저스토리 생략

### Changed
- 루트 유저스토리 지침 문장 — "유저스토리는 루트에 두지 않는다" → "루트는 화면 전체의 큰 목표, 영역은 세부 시나리오, 중복 금지"

## [1.3.0] - 2026-04-14

### Added
- `features[]` 필드 설명에 **"UI 구분 단위만"** 기준 명시. 기능명세에 있다고 전부 넣는 게 아니라 이 영역이 실제로 UI 영역으로 표현하는 기능만. `features[]`는 와이어프레임 `<bp-section>`과 1:1 대응. 맥락 rule은 description 본문 마크다운 링크로 참조
- 판단 기준 표: "UI 영역으로 표현" → features[] / "맥락·rule 참조만" → description 링크
- 자주 하는 실수에 "기능명세의 모든 feature를 `features[]`에 다 넣기" 항목 추가

## [1.2.0] - 2026-04-14

### Added
- `## 비고` 섹션 정의 추가 (루트/영역 본문 표 모두). "이 화면 리뷰 시 알아둘 점 — 화면 단위 고유 정보"이며 **와이어프레임의 `bp-description-note`로 그대로 옮겨지는 유일한 출처**임을 명시
- 자주 하는 실수에 "`## 비고`에 기능명세 rule 복붙" 항목 추가 — rule은 기능명세 SSOT, 비고는 화면 리뷰용 고유 정보만

## [1.1.0] - 2026-04-14

### Added
- 영역 파일(`area_*.md`)의 `features` 배열에 **빈 배열(`[]`) 금지** 규칙 명시. 영역과 기능명세 간 유일한 구조적 링크이므로 1개 이상 필수
- "왜 빈 배열이 금지인가" 설명 추가 — 빈 배열일 경우 기능명세 변경 영향을 추적할 수 없어 SSOT가 깨진다는 근거
- 자주 하는 실수에 두 항목 추가:
  - `area_*.md`의 `features: []` 빈 배열 금지
  - AC가 기능명세 rule을 그대로 베끼는 패턴 금지 (링크로 참조)

## [1.0.0] - 2026-04-14

타입별 파일 네이밍 컨벤션과 frontmatter 규약의 첫 정식 릴리스.

### Added
- 파일명 컨벤션 — `screen.md` / `sheet_{name}.md` / `dialog_{name}.md` / `area_{name}.md`
- frontmatter 필수 필드: `screenId`, `title`, `type`(`page`/`sheet`/`dialog`/`panel`), `purpose`, `viewport`
- title 접두사 규약: `화면_` / `시트_` / `다이얼로그_` / `영역_`
- 한 폴더에 페이지 1개 + 시트·다이얼로그·영역은 복수 가능
- 영역 파일(`area_*.md`)에 features 배열로 featureId + 기능명세 ref 매핑
- references 예제 3종: `screen.md`, `area_image.md`, `area_option.md`

### Changed
- references 파일명 갱신: `root_screens.md` → `screen.md`, `image_screens.md` → `area_image.md`, `option_screens.md` → `area_option.md`
- 화면명세에서 elements 위치를 영역 파일(`area_*.md`)로 통일 (페이지 화면명세는 레이아웃과 영역 링크만 담당)
- 유저스토리·인수조건은 해당 영역 파일에만 작성하고 페이지 루트에서는 생략

### Removed
- `label` 필드 폐기 (`title`로 통일, 카테고리 접두사로 종류 구분)
- 본문 ASCII 와이어프레임 섹션 (와이어프레임은 별도 `*.html` 파일이 SSOT)
