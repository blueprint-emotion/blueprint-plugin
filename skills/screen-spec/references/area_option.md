# 옵션 선택

## 유저스토리

- [Must] 구매자로서, 이 영역에서 옵션과 수량을 선택하고 장바구니에 담거나 바로 구매하고 싶다
- [Should] 구매자로서, 품절 옵션이 시각적으로 구분되길 원한다 — 헛된 시도를 피하기 위해
- [Could] 구매자로서, 옵션 조합별 미리보기 이미지를 보고 싶다

**elements**
- PRODUCT__OPTION 옵션을 단계별 셀렉트로 표시
  - 1차 옵션: 셀렉트
  - 2차 옵션: 셀렉트 (1차 선택 전 비활성)
- 수량 선택 (+/- 버튼)
- 선택 옵션 요약 (개별 삭제 가능)
- PRODUCT__PRICE 결제 금액을 실시간 합계로 표시
- 장바구니 담기 버튼
- 바로구매 버튼

## 인터랙션

| 구성요소 | 인터랙션 | 상세 |
|---|---|---|
| 1차 옵션 셀렉트 | 자동 포커스 | 화면 진입 시 첫 셀렉트에 포커스 (모바일 제외) |
| 수량 +/- 버튼 | 길게 누르기 | 500ms 이상 누르면 가속 증감 |
| 옵션 요약 칩 | 스와이프 (모바일) | 좌측 스와이프로 삭제 노출 |
| 바로구매 | 더블탭 차단 | 첫 탭 후 1초간 비활성화하여 중복 결제 방지 |

## 예외 케이스

| 상황 | 처리 방식 |
|---|---|
| 옵션 조회 API 실패 | 옵션 영역에 "불러오기 실패" + 재시도 버튼. 다른 영역은 영향 없음 |
| 선택한 옵션이 결제 직전 품절 전환 | 결제 시도 시 알림 모달 + 옵션 다시 선택 유도 |
| 수량이 재고 초과 | "최대 N개 구매 가능" 토스트 + 입력값 재고로 자동 보정 |
| 비로그인 + 장바구니 담기 | 로그인 화면으로 리다이렉트, 선택 옵션은 세션에 보존 |

## 인수조건

- [ ] 1차 옵션 선택 전 2차 옵션이 비활성 상태이다
- [ ] 품절 옵션은 취소선으로 표시되고 선택 불가하다
- [ ] 옵션·수량 변경 시 총 결제 금액이 실시간 갱신된다
- [ ] 선택한 옵션을 개별 삭제할 수 있다
- [ ] 결제 직전 품절 전환 시 결제가 차단된다
