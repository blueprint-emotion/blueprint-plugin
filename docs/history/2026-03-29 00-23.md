## 핵심 결정 사항 (Key Decisions)

- **3계층 구조**: `features/` (기능 단위 명세) + `screens/` (화면 = 레이아웃 + 기능 참조) + `wireframes/` (생성된 HTML). 기능과 화면을 분리하여 재사용 가능하게.
- **기능 분리 기준**: UI 요소 단위가 아닌 비즈니스 기능 단위 (독립 설명 가능, 재사용 가능, 하나의 개발 태스크)
- **에이전트 + 스킬 역할 분리**: spec 스킬(기획서 작성 도우미) + wireframe 스킬(HTML 생성/업데이트). 에이전트는 별도로 만들지 않고 스킬 2개로 구성.
- **자동 sync 배제**: md 수정마다 자동 와이어프레임 업데이트하지 않음. 사용자가 "와이어프레임 업데이트해줘" 명시적 요청 시에만 실행 (토큰 절약).
- **자연어 트리거 + 컨펌**: 사용자가 자연어로 뭘 고쳤는지 말하면 에이전트가 영향 범위 표시 → 사용자 컨펌 → 실행. 기획자는 git/파일경로 몰라도 됨.
- **2-pass 생성**: 최초 생성 시 screen md로 레이아웃 → feature md의 와이어프레임 요소만 순차 읽기. 업데이트 시 feature md만 읽고 HTML의 data-feature 영역 직접 교체 (screen md 안 거침).
- **usedIn 필드**: feature md 프론트매터에 이 기능을 쓰는 화면 목록. 업데이트 시 screen md 전수 스캔 없이 영향 범위 파악 가능.
- **spec 필드**: wireframe HTML 메타데이터에 `"spec": "../features/auth.md"` 추가하여 기능 명세 경로 연결.
- **flowframehq org**: GitHub org명. 사용자 설치 커맨드: `npx skills add flowframehq/wireframe-skill`
- **description optimization 루프 실패**: ANTHROPIC_API_KEY 환경변수 없어서 skill-creator의 run_loop.py 실행 불가. 수동으로 description 개선 완료.

## 시도와 결과 (Attempts & Results)

- `npx skills add ./` 로컬 테스트로 두 스킬 모두 정상 감지/설치 확인

### 실패한 것 (What Didn't Work)
- skill-creator의 description optimization 루프 (`python3 -m scripts.run_loop`): ANTHROPIC_API_KEY 없어서 실패. 향후 API 키 설정 후 재시도 가능.
- 첫 시도에서 `python` 명령 없음 (python3만 존재), anthropic 패키지 미설치 등 환경 이슈 발생.

## 다음 단계 (Next Steps)

### 1단계: 스킬 이슈 수정 (높음 우선)

아래 이슈를 스킬 SKILL.md에 직접 반영한다. **수정 후 다른 파일 정합성 작업 진행.**

**flowframe-wireframe SKILL.md 수정:**

1. **복잡한 레이아웃 가이드 추가** — `wf-page > wf-card` 외에 멀티 카드, 사이드바+메인+패널 등 복합 레이아웃 구성법. Base CSS에 관련 클래스가 있는지 확인 필요 (`/Users/jay/WebstormProjects/wireframe-skill/samples/assets/css/wireframe-base.css`).

2. **feature 없는 고정 영역 처리 추가** — screen md 레이아웃에서 feature 참조 없는 항목(상단 메뉴바, 하단 상태바 등)을 어떻게 렌더링할지 규칙 추가. 예: 직접 텍스트로 기술된 영역은 data-feature 없이 고정 요소로 렌더링.

3. **복수 feature 한 영역 배치 규칙** — 하나의 레이아웃 영역에 feature 여러 개가 들어갈 때 각 feature를 별도 data-feature 컨테이너로 순서대로 배치.

4. **feature ID 변환 규칙 명확화** — 와이어프레임 요소의 한국어 이름 → `{SCREEN_ID}_FEATURE_{ENGLISH_NAME}`. 에이전트가 한국어 요소명을 영어로 변환하여 ID 생성. 예: "서비스 로고" → `LOGO`, "댓글 목록" → `COMMENT_LIST`.

5. **업데이트 시 새 요소 추가 케이스** — feature md에 요소가 추가되면 HTML에서 해당 feature의 data-feature 섹션 끝에 새 요소 추가.

**flowframe-spec SKILL.md 수정:**

6. **삭제 워크플로우 추가** — "Removing a feature from a screen", "Deleting a feature entirely", "Deleting a screen" 워크플로우. usedIn 정리 포함.

7. **featureId 네이밍 규칙** — 한국어 기능명 → 영어 약칭 대문자. 예: "인증" → `AUTH`, "파일 업로드" → `FILE_UPLOAD`, "버전관리" → `VERSION_CONTROL`.

8. **정합성 검증 워크플로우** — "전체 정합성 확인해줘" 시: usedIn ↔ screen 참조 일치, feature 파일 존재 여부, wireframe과 spec 간 feature 매칭 확인.

**예시 정합성:**

9. **EXAMPLE.md의 auth 기능에 API 섹션 추가** (FEATURE-EXAMPLE.md와 일관성)
10. **SCREEN-EXAMPLE.md에 feature 없는 영역 처리 설명 추가**
