## Screen

### 레이아웃

1. 좌측 사이드바 — @FLOW/NODE, @FLOW/UPLOAD
   - 그룹별 노드 목록 (드래그로 그룹 간 이동 가능)
   - 하단에 플로우 업로드 버튼
2. 우측 상단 툴바 — @HISTORY/LIST, @HISTORY/LOAD, @VERSION/SAVE, @VERSION/NUMBERING, @HISTORY/RESET, @FLOW/LAYOUT, @HISTORY/UNDO
   - 버전 선택 셀렉트박스, 가져오기 버튼, 저장 버튼, 현재 버전 표시, 초기화 버튼, 정렬 버튼, 실행 취소 안내
3. 우측 메인 — @FLOW/NODE, @FLOW/EDGE
   - react-flow 캔버스 (노드 배치, 엣지 연결, 인라인 수정)
4. 모달: 플로우 업로드 — @FLOW/UPLOAD
   - 파일 선택, 형식 검증 결과, 미리보기 확인
5. 모달: 버전 저장 — @VERSION/SAVE
   - 덮어쓰기/메이저/마이너 옵션 선택, 버전 미리보기, 저장 실행

## Requirement

### 플로우 업로드 — @FLOW/UPLOAD
- Given 업로드 버튼이 표시된 상태 When 업로드 버튼 클릭 Then 파일 선택 모달이 열린다
- Given 모달이 열린 상태 When MD 파일을 선택 Then 형식 검증이 자동 실행된다
- Given 형식 검증 성공 When 미리보기 확인 버튼 클릭 Then 모달이 닫히고 캔버스에 플로우가 렌더링된다
- Given 형식 검증 실패 When 검증 결과 표시 Then 실패 사유 메시지가 표시되고 다른 파일을 재선택할 수 있다
- Given 파일 선택 전 When 미리보기 확인 버튼 확인 Then 미리보기 확인 버튼이 비활성 상태이다

### 노드 관리 — @FLOW/NODE
- Given 캔버스에 노드가 렌더링된 상태 When 노드 클릭 Then 해당 노드가 하이라이트되고 삭제 버튼이 표시된다
- Given 노드가 선택된 상태 When 노드 더블클릭 Then 인라인 수정 모드로 진입하고 입력 필드가 활성화된다
- Given 인라인 수정 모드 When Enter 키 입력 또는 포커스 이탈 Then 노드 이름이 업데이트된다
- Given 노드가 선택된 상태 When 삭제 버튼 클릭 Then 해당 노드와 연결된 엣지가 함께 삭제된다
- Given 좌측 사이드바에 노드 목록이 표시된 상태 When 노드 항목을 다른 그룹으로 드래그앤드롭 Then 해당 노드의 그룹이 변경된다

### 엣지 관리 — @FLOW/EDGE
- Given 캔버스에 엣지가 렌더링된 상태 When 엣지 클릭 Then 해당 엣지가 하이라이트되고 삭제 버튼이 표시된다
- Given 엣지가 선택된 상태 When 엣지 레이블 더블클릭 Then 인라인 수정 모드로 진입하고 입력 필드가 활성화된다
- Given 인라인 수정 모드 When Enter 키 입력 또는 포커스 이탈 Then 엣지 레이블이 업데이트된다
- Given 엣지가 선택된 상태 When 삭제 버튼 클릭 Then 해당 엣지가 삭제된다

### 레이아웃 정렬 — @FLOW/LAYOUT
- Given 캔버스에 노드와 엣지가 존재하는 상태 When 정렬 버튼 클릭 Then 오토 레이아웃 알고리즘이 실행되어 모든 노드와 엣지가 재배치된다

### 버전 저장 — @VERSION/SAVE
- Given 저장 버튼 클릭 When 저장 모달이 열림 Then 덮어쓰기, 메이저 버전 올리기, 마이너 버전 올리기 옵션이 표시된다
- Given 최초 저장 시 When 저장 모달이 열림 Then 메이저 버전 올리기만 선택 가능하고 버전 미리보기에 "1.0"이 표시된다
- Given 저장 이력이 있는 상태 When 저장 모달이 열림 Then 세 가지 옵션 모두 활성이고 덮어쓰기가 기본 선택된다
- Given 옵션을 선택한 상태 When 다른 옵션으로 변경 Then 버전 미리보기에 저장될 버전 번호가 즉시 반영된다
- Given 옵션을 선택한 상태 When 저장 버튼 클릭 Then 선택한 옵션으로 버전이 저장되고 모달이 닫힌다
- Given 저장 모달이 열린 상태 When 취소 버튼 클릭 또는 모달 외부 클릭 Then 모달이 닫히고 변경 없음

### 버전 번호 표시 — @VERSION/NUMBERING
- Given 저장된 버전이 없는 상태 When 화면 로드 Then 버전 표시 영역에 "미저장" 텍스트가 표시된다
- Given 버전이 저장된 상태 When 화면 로드 또는 저장 완료 Then 현재 버전 번호가 표시된다 (예: "v1.0")

### 버전 목록 — @HISTORY/LIST
- Given 저장된 버전이 존재하는 상태 When 셀렉트 박스 클릭 Then 저장된 버전 목록이 펼쳐진다
- Given 저장된 버전이 없는 상태 When 셀렉트 박스 클릭 Then 빈 목록 안내 텍스트가 표시된다
- Given 버전 목록이 펼쳐진 상태 When 다른 버전을 선택 Then 선택된 버전이 표시되고 가져오기 버튼이 노출된다

### 버전 가져오기 — @HISTORY/LOAD
- Given 버전 목록에서 버전을 선택한 상태 When 가져오기 버튼 클릭 Then 선택한 버전의 데이터가 캔버스에 로드된다
- Given 버전이 선택되지 않은 상태 When 가져오기 버튼 확인 Then 버튼이 비활성 상태이다

### 실행 취소 — @HISTORY/UNDO
- Given 편집 히스토리에 되돌릴 단계가 있는 상태 When Cmd+Z 입력 Then 편집 히스토리에서 한 단계 되돌린다
- Given 되돌릴 단계가 없는 상태 When Cmd+Z 입력 Then 아무 동작이 발생하지 않는다

### 초기화 — @HISTORY/RESET
- Given 편집 변경사항이 존재하는 상태 When 초기화 버튼 클릭 Then "모든 변경사항이 사라집니다. 초기화하시겠습니까?" 확인 다이얼로그가 표시된다
- Given 확인 다이얼로그가 표시된 상태 When 확인 클릭 Then 캔버스가 초기 상태(또는 마지막 로드한 버전)로 복원된다
- Given 확인 다이얼로그가 표시된 상태 When 취소 클릭 Then 다이얼로그가 닫히고 현재 상태가 유지된다
- Given 변경사항이 없는 상태 When 초기화 버튼 확인 Then 초기화 버튼이 비활성 상태이다

## UserStory

### 플로우 업로드 — @FLOW/UPLOAD
- 기획자로서 MD 파일을 업로드하여 플로우를 시각화하고 싶다, 작성한 기획 문서를 시각적으로 확인하기 위해
- 기획자로서 업로드 시 형식 검증 결과를 확인하고 싶다, 올바른 형식의 파일만 캔버스에 반영하기 위해

### 노드 관리 — @FLOW/NODE
- 기획자로서 캔버스에서 노드 이름을 수정하고 싶다, 플로우의 내용을 빠르게 변경하기 위해
- 기획자로서 불필요한 노드를 삭제하고 싶다, 플로우를 정리하기 위해
- 기획자로서 노드를 다른 그룹으로 이동하고 싶다, 플로우의 구조를 재구성하기 위해

### 엣지 관리 — @FLOW/EDGE
- 기획자로서 엣지 레이블을 수정하고 싶다, 노드 간 관계를 명확하게 표현하기 위해
- 기획자로서 불필요한 엣지를 삭제하고 싶다, 플로우의 연결 관계를 정리하기 위해

### 레이아웃 정렬 — @FLOW/LAYOUT
- 기획자로서 노드와 엣지를 자동 정렬하고 싶다, 복잡한 플로우를 보기 좋게 배치하기 위해

### 버전 저장 — @VERSION/SAVE
- 기획자로서 현재 플로우를 버전별로 저장하고 싶다, 작업 이력을 관리하기 위해
- 기획자로서 저장 시 버전 번호 방식을 선택하고 싶다, 변경의 중요도에 따라 버전을 구분하기 위해

### 버전 번호 표시 — @VERSION/NUMBERING
- 기획자로서 현재 작업 중인 버전 번호를 확인하고 싶다, 어느 버전을 편집하고 있는지 파악하기 위해

### 버전 목록 — @HISTORY/LIST
- 기획자로서 저장된 버전 목록을 확인하고 싶다, 과거 버전을 선택하여 비교하기 위해

### 버전 가져오기 — @HISTORY/LOAD
- 기획자로서 과거 버전을 현재 캔버스에 불러오고 싶다, 이전 상태로 돌아가거나 비교하기 위해

### 실행 취소 — @HISTORY/UNDO
- 기획자로서 직전 편집을 되돌리고 싶다, 실수한 변경을 즉시 취소하기 위해

### 초기화 — @HISTORY/RESET
- 기획자로서 모든 변경사항을 초기 상태로 되돌리고 싶다, 처음부터 다시 편집하기 위해
