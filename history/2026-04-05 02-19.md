## 핵심 결정 사항 (Key Decisions)

- **DOM fragment 이동 패턴**: wrapper 컴포넌트는 `html(this)` 대신 `DocumentFragment`로 자식을 이동. ES module의 define 순서 때문에 안쪽 컴포넌트가 먼저 렌더링되어 html(this)로 캡처하면 이미 렌더링된 HTML을 다시 직렬화→재파싱해서 이중 렌더링 발생
- **`__bp` 렌더링 가드**: bp-core의 `define()` 함수에서 connectedCallback을 래핑하여 `this.__bp` 플래그로 이중 호출 방지. fragment 이동 시 disconnect→reconnect 사이클에서 재렌더링 차단
- **bp-page `description` 속성**: `<bp-page description>` → aside 패널 노출 (개발용), `<bp-page>` → 패널 없이 (플랫폼용). 플랫폼은 blueprint-meta JSON을 메타태그로 읽어서 자체 UI 생성
- **bp-page header/footer 슬롯**: `slot="header"` / `slot="footer"` → 자동 분배. `data-slot="bp-page-content"` 안에 bp-page-body(header+main+footer) 구조. 플랫폼에서 bp-page-content의 innerHTML만 가져감
- **메타바 디자인**: 라운드 없음, 바텀 라인만, 상단 sticky, 가로 나열(flex-wrap), `·` 구분자
- **디스크립션 패널**: 컴포넌트 1개당 메타 1개, description에 `\n`으로 여러 줄, 카드형 레이아웃(border + rounded-lg), hover시 border-primary/40 + ring-primary/20
- **호버 하이라이트 양방향**: 디스크립션→와이어프레임(outline 3px primary/20%, offset 5px), 와이어프레임→디스크립션(Tailwind class 토글 border-primary/40 ring-1 ring-primary/20), 와이어프레임 자체 호버 시에도 자신에 outline 표시
- **toggle/toggle-group 삭제**: 와이어프레임에서 불필요하다고 판단하여 컴포넌트 자체 삭제
- **sidebar 쇼케이스 제거**: 전체 레이아웃을 잡는 컴포넌트라 인라인 쇼케이스에 부적합. bp-page에 커스텀으로 구현 예정
- **플랫폼 배포**: bp-components.js + base.css를 CDN 호스팅, 플러그인 패키지에는 스킬/에이전트만 포함
- **`<p>` 태그 금지**: 웹 컴포넌트의 innerHTML 교체 시 DOM 자동 보정 문제로 `<div>` 또는 `<span>` 사용
- **header/footer 스타일**: `text-muted-foreground text-xs`로 존재감 최소화, border는 bp-page-body가 자동 적용

## 주의사항 (Caveats)

- **브랜치**: `components` 브랜치에서 작업 중. main에 머지 전
- **빌드 명령어**: `npx --yes esbuild components/index.ts --bundle --format=esm --outfile=components/bp-components.js --target=es2022`
- **Tailwind CDN 로드 순서**: HTML에서 반드시 `base.css` → 컴포넌트 JS → `<style type="text/tailwindcss">` (theme) → Tailwind CDN 순서
- **`hidden` 클래스 금지**: 토글이 필요한 요소에 Tailwind `hidden` 대신 `style="display:none"` 사용
- **`<p>` 태그 금지**: 웹 컴포넌트 내부에서 `<p>` 사용 금지. DOM 자동 보정으로 구조 깨짐
- **오버레이 컴포넌트 trigger 속성**: dialog, alert-dialog, drawer, sheet, popover, sonner — `trigger` 속성 필수로 버튼 클릭 방식 사용
- **bp-separator vertical**: 호스트에 `display:flex`를 설정. class로 높이 커스텀 가능 (예: `class="h-5"`). h- 클래스 없으면 `h-full` 기본
- **toggle/toggle-group 삭제됨**: 파일 삭제 + index.ts에서 import 제거 완료
- **sidebar 쇼케이스 미포함**: 별도 페이지 또는 bp-page 내부 커스텀으로 테스트 필요
- **http 서버**: `cd components && python3 -m http.server 8765`로 로컬 확인 (watch는 터미널에서 별도 실행)
