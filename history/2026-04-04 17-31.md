## 목표 (Goal)

기존 와이어프레임의 인라인 Tailwind HTML을 `bp-*` 웹 컴포넌트로 교체하여, wireframer 에이전트의 토큰 사용량을 1/3로 줄이고 디자인 일관성을 강제한다. shadcn/ui 디자인 시스템(oklch 컬러)을 그대로 사용한다.

## 핵심 결정 사항 (Key Decisions)

- **shadcn 클래스를 정확히 복사**: 1차 시도에서 에이전트가 `rounded-none`을 자의적으로 넣어서 전부 삭제 후 재작업. 새 `.shadcn/ui/` 소스가 업데이트되어 있었고(rounded-md, bg-input/20 등), 이 소스의 클래스를 그대로 사용해야 함
- **커스텀 엘리먼트 접두어 `bp-`**: HTML 표준상 하이픈 필수, `bp-` 접두어 채택
- **Tailwind Browser CDN v4 환경**: `@theme inline` 블록을 `<style type="text/tailwindcss">`에, CSS 변수는 `base.css`에 분리. 로드 순서: base.css → 컴포넌트 JS → Tailwind CDN (마지막)
- **`hidden` 대신 `style="display:none"`**: 컴포넌트가 Tailwind CDN보다 먼저 렌더링되므로 `hidden` 유틸리티가 안 먹음. 토글이 필요한 모든 컴포넌트에서 inline style 사용
- **오버레이 컴포넌트 이중 모드**: `trigger` 속성이 있으면 버튼+모달/시트 동작, 없으면 인라인 미리보기 (와이어프레임 내 사용)
- **`data-feature`/`data-el`은 웹 컴포넌트에 직접 사용 가능**: HTML 표준 data-* 속성이라 커스텀 엘리먼트에도 자동으로 붙음, 별도 div 래핑 불필요
- **폼 상태/이벤트 불필요**: 와이어프레임 컴포넌트라 checked/value 동기화, change 이벤트 dispatch 등은 과도한 설계로 판단하여 미구현

## 시도와 결과 (Attempts & Results)

### 성공한 것
- 에이전트 10개 병렬 실행으로 50개 컴포넌트 동시 생성 (약 2분)
- esbuild 번들링 — npx로 설치 없이 사용, 14ms 빌드
- `connectedCallback` + `innerHTML` 패턴으로 모든 컴포넌트 통일
- trigger 모드의 dialog/sheet/drawer — 실제 오버레이 동작이 와이어프레임에서도 가능

### 실패한 것
- **1차 병렬 생성**: 에이전트가 shadcn 소스의 `rounded-none`을 그대로 복사 → 단독 사용 시 라운드 없이 렌더링. 전부 삭제 후 새 shadcn 소스 기반으로 재작업
- **`hidden` 클래스 사용**: Tailwind Browser CDN이 처리하기 전에 컴포넌트가 렌더링되어 초기에 콘텐츠가 보임. `style="display:none"`으로 전환
- **3개 에이전트 API 리밋**: 10개 병렬 중 3개(E, F, J)가 리밋에 걸렸으나, 파일 작성은 이미 완료된 상태여서 실제 손실 없음 (bp-image만 수동 생성)

## 주의사항 (Caveats)

- **브랜치**: 현재 `components` 브랜치에서 작업 중. main에 머지 전
- **빌드 명령어**: `npx --yes esbuild components/index.ts --bundle --format=esm --outfile=components/bp-components.js --target=es2022`
- **Tailwind CDN 로드 순서**: showcase.html에서 반드시 `base.css` → 컴포넌트 JS → `<style type="text/tailwindcss">` (theme) → Tailwind CDN 순서
- **`hidden` 클래스 금지**: 토글이 필요한 요소에 Tailwind `hidden` 클래스 쓰면 안 됨. `style="display:none"` 사용할 것
- **shadcn 소스가 업데이트됨**: `.shadcn/ui/` 폴더의 tsx 파일들은 사용자가 새로 넣은 버전. 이전 버전과 클래스가 다르므로 반드시 현재 소스를 읽고 작업할 것
- **bp-page 미검증**: 좌우 분할 레이아웃 + 메타데이터 자동 파싱은 아직 실제 와이어프레임에서 테스트 안 됨
- **`<p>` 태그 절대 금지**: showcase.html과 컴포넌트 내부에서 `<p>` 태그 사용 금지. 웹 컴포넌트의 `innerHTML` 교체 시 `<p>` 안에 block 요소가 들어가면 브라우저가 DOM을 자동 보정하여 구조가 깨짐. `<div>` 또는 `<span>`을 사용할 것
