## 핵심 결정 사항 (Key Decisions)

- **planning-workflow는 하네스 스킬로 유지**: `context: fork` + `agent:` 패턴이 공식 문서에서 지원됨을 확인. 에이전트 전환이나 자연 라우팅 대신 명시적 오케스트레이션 선택
- **reviewer는 읽기 전용 판정 에이전트**: Read/Grep/Glob만 사용, hook agent(`type: "agent"`) 호환. pass/fail 판정만 반환하고 수정은 planner/wireframer가 수행
- **references는 스킬당 CLAUDE.md(자동 로드 TOC) + 디테일한 예제 2개**: 소수 정예 전략. CLAUDE.md가 자동 로드되어 에이전트가 어떤 예제를 참조할지 안내
- **와이어프레임은 "캔버스 위의 화면"**: 풀블리드 앱이 아님. `body`에 `bg-zinc-100 p-8`, 화면 컨테이너에 `rounded-xl border shadow-sm`
- **화면 프레임에 `overflow-hidden` 금지**: outline 호버가 잘림. 대신 header에 `rounded-t-xl`, footer에 `rounded-b-xl`
- **`min-w-[1280px]`**: max가 아닌 min으로 설정하여 화면이 최소 폭 보장
- **고정 영역(헤더/푸터)은 기본 라벨만**: "Header" / "Footer" 텍스트만 표시, 커스텀 필요 시에만 내용 추가
- **호버 CSS — 인디고 3px/3px**: `outline: 3px solid rgba(99, 102, 241, 0.55); outline-offset: 3px; border-radius: 6px`. data-el 호버에 background-color 없음 (검정 버튼 위에서 흰색 문제)
- **리뷰 탭은 플랫폼 JS로 전환**: CSS-only 보일러플레이트 제거. `data-state="상태명"` 속성만 붙이면 `flowframe.js`가 자동으로 리뷰 탭 생성. 와이어프레임 HTML이 극적으로 단순화됨
- **이미지 플레이스홀더**: `aspect-[4/3]` (aspect-square 금지), `bg-zinc-100 dark:bg-zinc-800` (배경보다 미묘하게 연한 톤)

## 시도와 결과 (Attempts & Results)

### 성공한 것
- flowframe.js가 `data-state` 기반으로 리뷰 탭을 자동 생성하고 상태 전환도 정상 동작
- 호버 하이라이트도 JS로 CSS 주입하여 와이어프레임 `<style>` 블록에서 제거 가능
- 화면 프레임 규격(캔버스+컨테이너)으로 와이어프레임 외관이 크게 개선
- `overflow-hidden` 제거 + header/footer 개별 둥근 모서리로 outline 잘림 문제 해결

### 실패/재고한 것
- 처음에 `outline-offset: -2px` (inset)으로 잘림 방지 시도 → 동작하지만 outline이 안쪽으로 들어가 예쁘지 않음 → `overflow-hidden` 제거가 근본 해결
- data-el 호버에 `background-color` 오버레이 → 검정 버튼 위에서 흰색으로 변하는 문제 → 제거

## 주의사항 (Caveats)

- **flowframe.js는 `data-state`를 `[data-feature]`의 직접 자식에서만 찾음**: `:scope > [data-state]` 셀렉터 사용. 중첩된 data-state는 무시됨
- **SKILL.md와 references에 아직 구 CSS-only 리뷰 탭 코드가 남아있음**: 다음 단계 1~4에서 일괄 교체 필요. 현재 상태에서 wireframer가 생성하면 구 방식으로 생성할 수 있음
- **test.html은 프로젝트 루트에 있음**: 작업용 테스트 파일. 최종 정리 시 삭제 또는 이동 검토
- **`.claude/worktrees/`에 오래된 worktree 3개 남아있음**: `flamboyant-rubin`, `kind-mestorf`, `modest-hopper` — 구 구조 파일들이 있어 혼동 가능. 정리 검토
- **호버 CSS는 flowframe.js가 주입**: 와이어프레임 HTML의 `<style>` 블록에는 Tailwind 다크모드 설정(`@custom-variant dark`)만 남음
