# 상품 상세

## 유저스토리

- [Must] 구매자로서, 이 화면을 통해 상품 정보를 종합적으로 확인하고 구매 결정을 내리고 싶다

> 영역별 세부 시나리오는 각 area_*.md에. 루트는 화면 전체의 큰 목표만.

## 화면 구성

상단은 2컬럼 그리드(1:1)로 구성한다.

- **좌측**: [이미지 갤러리](./area_image.md)
- **우측 상단**: [상품 기본 정보](./area_info.md)
- **우측 하단**: [옵션 선택](./area_option.md)

하단은 탭 영역으로, "상세정보"와 "Q&A" 탭을 전환한다.

- **Q&A 탭**: Q&A 목록 + 문의 작성 버튼

## 상태

| 상태 | 표시 |
|------|------|
| 로딩 중 | 전체 영역 스켈레톤 |
| 정상 | 상품 정보 + 이미지 + 옵션 + 탭 영역 |
| 상품 없음 | "상품을 찾을 수 없습니다" 안내 |
| 품절 | 정상 표시 + 구매 버튼 비활성화 + "일시품절" |

## 예외 케이스

| 상황 | 처리 방식 |
|---|---|
| 상품 API 500 응답 | "일시적 오류" 토스트 + 재시도 버튼 |
| 네트워크 오프라인 | 오프라인 배너 + 캐시된 마지막 데이터 유지 |
| 상품 ID 없음 (404) | "상품을 찾을 수 없습니다" 화면으로 전환 |
| 비로그인 + 문의하기 | 로그인 화면으로 리다이렉트, 현재 경로를 쿼리로 보존 |

## 인수조건

- [ ] Q&A 탭은 탭 전환 시 lazy load된다
- [ ] 비로그인 상태에서 "문의하기" 클릭 시 로그인 페이지로 이동한다
- [ ] 이미지 갤러리와 상품 정보 영역은 동시에 로딩한다 (병렬 fetch)
